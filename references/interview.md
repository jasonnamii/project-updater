# 인터뷰 프로토콜

발동 즉시 실행. 유형 판별(1단계) → 정보 수집(2단계) → 초안 생성으로 이어진다.

---

## 1단계: 유형 판별 (공통)

AskUserQuestion으로 한 번에 묻는다. 3개 질문 동시.

```
Q1. 이 프로젝트의 성격은? (유형 판별)
  - 외부 의뢰인의 프로그램/서비스를 설계한다 → A
  - 리서치→기획→산출물 파이프라인이다 → B
  - 단건 분석/판단이다 → C
  - 기타 (직접 설명)

Q2. 프로젝트 폴더가 이미 있나?
  - 있다 (폴더 마운트 해줄게) → 폴더 스캔 진행
  - 없다 (새로 만들어야 함) → 폴더 생성 제안
  - 모르겠다

Q3. 상위 프로젝트 그룹이 있나?
  - C8 (Cre8orClub) 계열 → 00 C8 공통/CLAUDE.md 참조 삽입
  - 피디님 프로젝트 계열
  - 독립 프로젝트
  - 기타 (직접 설명)
```

---

## 2단계: 유형별 정보 수집

유형 확정 후, 해당 유형의 질문 세트를 AskUserQuestion + 자유 입력으로 수집한다.

### 유형 A: 풀 스펙 (7개 질문)

| # | 질문 | 채워지는 섹션 | 필수 |
|---|------|-------------|------|
| A1 | 프로젝트를 한 문장으로 설명하면? (사업명+핵심 포지션) | §1 좌표계 | ✅ |
| A2 | 의뢰인은 누구이고, 핵심 의사결정자는? (이름+역할) | §2 인물 | ✅ |
| A3 | 타겟(고객/참가자)은 누구인가? | §1 좌표계 | ✅ |
| A4 | 수익 모델은? (어떻게 돈을 버는지) | §3 전략 | ✅ |
| A5 | 유사한 기존 프로젝트가 있나? (있으면 차이점 중심으로) | §4 규칙 | 선택 |
| A6 | 이미 만들어진 문서/자료가 있나? (있으면 폴더 스캔으로 대체) | §5 SoT | 선택 |
| A7 | 확정된 핵심 수치가 있나? (가격, 인원, 예산, 일정 등) | §6 수치 | 선택 |

**A1~A4 수집 완료 시 초안 생성 가능.** A5~A7은 있으면 품질↑, 없어도 `[미확정]`으로 진행.

### 유형 B: 표준형 (4개 질문)

| # | 질문 | 채워지는 섹션 | 필수 |
|---|------|-------------|------|
| B1 | 이 프로젝트의 목적은? (무엇을 만들거나 판단하는지) | §1 상태 | ✅ |
| B2 | 현재 어디까지 진행됐나? (Phase 0~3 중) | §1 상태 | ✅ |
| B3 | 지켜야 할 특수 규칙이 있나? (SoT 취급, 편향방지, 용어 등) | §2 규칙 | 선택 |
| B4 | 확정된 핵심 수치가 있나? (예산, 규모, 일정 등) | §4 수치 | 선택 |

**B1~B2 수집 완료 시 초안 생성 가능.**

### 유형 C: 최소형 (3개 질문)

| # | 질문 | 채워지는 섹션 | 필수 |
|---|------|-------------|------|
| C1 | 무엇을 분석/판단하는 프로젝트인가? | §1 상태 | ✅ |
| C2 | 현재 결론이 나왔나, 진행 중인가? | §1 상태 | ✅ |
| C3 | 특별히 주의할 규칙이 있나? | §2 규칙 | 선택 |

**C1~C2 수집 완료 시 초안 생성 가능.**

---

## 실행 지침

1. **1단계는 AskUserQuestion 1회로 3개 질문 동시 수집.** 왕복 최소화.
2. **2단계는 필수 질문만 AskUserQuestion으로, 나머지는 자유 입력 안내.** "추가로 알려줄 것이 있으면 자유롭게"로 마무리.
3. **폴더가 마운트되어 있으면 A6/B3/C3는 스캔으로 대체** — 형에게 묻지 않고 파일 목록에서 추출.
4. **필수 질문 답변이 모이면 즉시 초안 생성.** 선택 질문 미답변 = `[미확정]`으로 진행.
5. **형이 "그냥 빨리 만들어"라고 하면** → 폴더 스캔 결과 + 최소 추론으로 C유형 초안 즉시 생성. 나중에 updater로 보강.
