# GRAPH.md 지식그래프 생성·갱신 프로토콜

**핵심 원칙: GRAPH.md는 상위 기획 + 핵심 엔티티 노드 전용.** 디테일(실행메모·일일로그·드래프트)은 노드가 아니라 엔티티의 참조 링크로만 등장. (SKILL.md INVARIANT 6)

```
①폴더 스캔 → ②엔티티 추출(5카테고리 필터) → ③그래프 초안 생성 → ④형 검토·수정 → ⑤저장
```

---

## ① 폴더 스캔

프로젝트 폴더의 전체 파일 목록 수집. .md 파일 우선, PDF·기타는 파일명만.

**결과물:** 파일 인벤토리 (파일명·크기·수정일). 이 단계에서 노드 확정 ❌. 엔티티 추출 재료로만 사용.

---

## ② 엔티티 추출 — 5 카테고리 필터

**문서가 노드가 아니다. 엔티티가 노드다.** 문서를 읽되, 다음 5 카테고리 엔티티를 식별·추출한다.

### 📋 상위 기획 (Planning)
**포함:** 비전·미션·전략·청사진·로드맵·핵심 의사결정·합의안·통합전략
**제외:** 실행 태스크·진행 메모·드래프트·중간 산출물
**판정 질문:** "이 내용이 프로젝트의 방향성·뼈대를 정의하는가?"

### 🎯 핵심 고려사항 (Considerations)
**포함:** 의사결정 기준·제약조건·원칙·리스크·전제·핵심 트레이드오프
**제외:** 일반 메모·지나가는 주석·일시적 관심사
**판정 질문:** "이것이 의사결정 때 반복적으로 참조되는가?"

### 👤 핵심 인물 (People)
**포함:** 의사결정권자·이해관계자·핵심 전문가·멘토·키 파트너 개인
**제외:** 단순 언급 인물(예: "A도 안다"), 일회성 접촉자
**판정 질문:** "이 인물의 입장·판단이 프로젝트에 영향을 주는가?"

### 🏢 핵심 조직 (Organizations)
**포함:** 파트너사·경쟁사·규제기관·커뮤니티·투자자·고객사
**제외:** 단순 참조 조직, 배경 설명용 조직
**판정 질문:** "이 조직과의 관계·입장이 프로젝트에 영향을 주는가?"

### 🌐 핵심 상황 (Situations)
**포함:** 시장환경·규제환경·타이밍·기술환경·외부 조건·결정적 컨텍스트
**제외:** 일시적 상태·단발 이벤트·일반 배경설명
**판정 질문:** "이 상황이 의사결정에 영향을 주는 외부 조건인가?"

### 추출 기법
- 각 .md 파일 읽기 → 5 카테고리 해당 엔티티 식별
- **대형 폴더(문서 20개+)**: Agent tool로 병렬 읽기 권장
- 동일 엔티티가 여러 문서에 등장 → 하나의 노드로 통합, 참조 문서를 모두 링크
- 애매한 경우 → 상위 기획 우선 → 배제 방향

**인터뷰 보조:** 사용자에게 "핵심 인물/조직/상황이 문서에 명시되지 않았다면 직접 알려달라"고 요청 가능. 형의 암묵지를 그래프로 외재화.

## GRAPH.md 갱신

| 트리거 | 행동 |
|--------|------|
| "GRAPH.md 갱신" 요청 | 기존 읽기 → 폴더 재스캔 → 엔티티 재추출 → 신규/변경/삭제 식별 → diff 반영 |
| 새 엔티티 감지 (새 인물·조직·상황·전략 등장) | "GRAPH.md 갱신 필요: 엔티티 N개 추가" 1줄 제안 (자동 실행 안 함) |
| 기존 엔티티 상태 변경 | 해당 엔티티 노드만 외과적 수정 |
| 디테일 문서만 추가/변경 | **갱신 불필요** (INVARIANT 6 — 디테일은 노드 아님) |

**갱신 규칙:** 기존 엔티티 의의·관계 유지, 변경분만 외과적 반영. frontmatter `updated` + `node-count` 재계산.
