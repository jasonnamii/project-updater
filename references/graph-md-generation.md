# GRAPH.md 지식그래프 생성·갱신 프로토콜

```
①폴더 스캔 → ②문서 읽기(내용 분석) → ③그래프 초안 생성 → ④형 검토·수정 → ⑤저장
```

## ① 폴더 스캔

프로젝트 폴더의 전체 파일 목록 수집. .md 파일 우선, PDF·기타는 파일명만.

## ② 문서 읽기

**각 .md 파일의 내용을 실제로 읽는다.** (CLAUDE.md 생성의 폴더 스캔과 핵심 차이 — 파일명이 아닌 내용 분석)
- 문서별 3-5줄 요약 추출
- 핵심 개념·용어 식별
- 문서 간 관계 파악 (상위/하위, 근거, 참조, 선후, 통합)

**대형 폴더 (문서 20개+)**: Agent tool로 병렬 읽기 권장.

## ③ 그래프 초안 생성

`→ graph-template.md 참조`

| 규칙 | 이유 |
|------|------|
| Obsidian `[[위키링크]]` 사용 | 그래프뷰 연동 + LLM 컨텍스트 효율 |
| 문서별 요약 3-5줄 + 핵심개념 + 상태 + 관계 | 풀스펙 |
| 관계 맵 섹션으로 전체 조감도 | 개별 노드 + 조감도 이중 제공 |
| 미해결·갭 섹션 필수 | BLIND_SPOT 원칙 |
| 파일명: `GRAPH.md` 고정 | 프로젝트 폴더 루트 |

**경량 모드 (문서 50개+)**: 요약 2줄 축약, 핵심 문서만 풀스펙. node-count > 50이면 자동 제안.

## ④ 형 검토 → ⑤ 저장

형 승인 후 저장.

## GRAPH.md 갱신

| 트리거 | 행동 |
|--------|------|
| "GRAPH.md 갱신" 요청 | 기존 읽기 → 폴더 재스캔 → 신규/삭제/변경 식별 → diff 반영 |
| 문서 추가/삭제 감지 | "GRAPH.md 갱신 필요" 1줄 제안 (자동 실행 안 함) |
| CLAUDE.md 구조 변경 | 프로젝트 개요 섹션 갱신 필요 여부 확인 |

**갱신 규칙:** 기존 요약·관계 유지, 변경분만 외과적 반영. frontmatter `updated` + `node-count` 재계산.

## 체크리스트 (GRAPH.md 생성 후)

- [ ] 모든 주요 .md 파일이 노드로 등록되었는가
- [ ] `[[위키링크]]`가 실제 파일명과 일치하는가

- [ ] 관계 맵이 문서 노드의 관계와 일치하는가
- [ ] 미해결·갭 섹션이 있는가
- [ ] frontmatter node-count가 실제 노드 수와 일치하는가
- [ ] 프로젝트 폴더 루트에 저장되었는가
