# GRAPH.md 표준 템플릿 + 작성 가이드

GRAPH.md 초안 생성 시 이 템플릿을 로드하여 사용한다.

---

## 템플릿

```markdown
---
type: knowledge-graph
project: "{프로젝트명}"
updated: "YYYY-MM-DD"
node-count: N
---

# {프로젝트명} 지식그래프

> [!info] 이 문서의 용도
> 이 프로젝트 폴더를 마운트하고 질문하면, 이 그래프를 기반으로 답변하거나 관련 문서를 안내합니다.

## 프로젝트 개요
{프로젝트의 핵심 목적·배경 2-3문장. LLM이 이것만 읽고도 "이 프로젝트가 뭔지" 즉시 파악 가능해야 함}

## 핵심 개념
- **{개념A}**: {정의 1-2문장}. 관련 → [[문서1]], [[문서2]]
- **{개념B}**: {정의 1-2문장}. 관련 → [[문서3]]

## 문서 노드

### 📄 [[문서명_v버전]]
- **요약**: {3-5줄 상세 요약. 경량 모드: 2줄}
- **핵심개념**: {개념A}, {개념B}
- **상태**: 확정 | 진행중 | 아카이브
- **관계**:
  - → 상위: [[상위문서]]
  - ← 근거: [[리서치문서]]
  - ↔ 참조: [[관련문서]]

## 관계 맵
- [[문서A]] →(근거제공)→ [[문서B]]
- [[문서B]] →(상위전략)→ [[문서C]]
- [[문서D]] ↔(상호참조)↔ [[문서E]]

## 하위 폴더 구조
- `_archive/` — 과거 버전·폐기 문서
- `_research/` — 리서치 자료
- `_sources/` — 원본 소스
- `PDFs/` — PDF 파일

## 미해결·갭
- {아직 문서화되지 않은 영역이나 의사결정 대기 사항}
```

## 작성 가이드

### 각 섹션의 역할

| 섹션 | 역할 | 필수 |
|------|------|------|
| frontmatter | 메타데이터. `node-count`로 규모 파악 | ✅ |
| 프로젝트 개요 | LLM의 1차 필터. 질문의 관련성 판단 | ✅ |
| 핵심 개념 | 프로젝트 고유 용어 사전. 질문 매칭 → 관련 문서 안내 | ✅ |
| 문서 노드 | 각 문서의 풀스펙. 위키링크로 그래프뷰 연동 | ✅ |
| 관계 맵 | 전체 조감도. 문서 노드의 관계를 한눈에 | ✅ |
| 하위 폴더 구조 | 서브디렉토리 역할 설명 | 폴더 있을 때 |
| 미해결·갭 | BLIND_SPOT. 빠진 것 명시 | ✅ |

### 관계 유형 표기

| 표기 | 의미 | 예시 |
|------|------|------|
| →(근거 제공)→ | A가 B의 근거/출처 | 리서치 → 기획안 |
| →(상위)→ | A가 B의 상위 문서 | 로드맵 → 액션리스트 |
| →(cascade)→ | A 변경 시 B 연쇄 변경 | 청사진 → 액션리스트 |
| →(통합)→ | A가 B에 통합됨 | 리서치+전략 → 통합본 |
| →(후속)→ | A 이후 B가 작성됨 | 통화정리 v1.0 → v2.0 |
| ↔(참조)↔ | 상호 참조 관계 | 기획안 ↔ 분석리포트 |
| ↔(병렬)↔ | 동등한 대안 관계 | 비전A ↔ 비전B |

### 경량 모드 (문서 50개+)

- 요약을 2줄로 축약
- 핵심 문서(현행 마스터, 의사결정 근거)만 풀스펙
- 나머지는 1줄 요약 + 관계만
- frontmatter에 `mode: lightweight` 추가

### 위키링크 규칙

- 파일명에서 `.md` 제외: `[[자벌레_통합전략_v1.0]]` ✅, `[[자벌레_통합전략_v1.0.md]]` ❌
- 파일명 정확 일치 필수. 추측으로 링크 생성 금지
- 하위 폴더 파일은 경로 포함: `[[_archive/구버전]]`
- PDF는 위키링크 대신 파일명만 텍스트로 기재

### 갱신 시 diff 규칙

- 신규 문서 → 노드 추가 + 관계 맵 갱신
- 삭제 문서 → 노드 제거 (아카이브 이동 시 상태를 "아카이브"로)
- 버전 업 → 요약·상태·관계 갱신, 구버전 노드 제거
- 변경 없는 문서 → 요약·관계 그대로 유지 (재작성 금지)
- frontmatter `updated` + `node-count` 재계산
