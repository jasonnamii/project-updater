# GRAPH.md 표준 템플릿 + 작성 가이드

GRAPH.md 초안 생성 시 이 템플릿을 로드하여 사용한다.

**철칙: 노드 = 엔티티. 노드 ≠ 문서.** 문서는 엔티티의 참조 링크로만 등장.

---

## 템플릿

```markdown
---
type: knowledge-graph
project: "{프로젝트명}"
updated: "YYYY-MM-DD"
node-count: N
categories: [planning, considerations, people, organizations, situations]
---

# {프로젝트명} 지식그래프

> [!info] 이 문서의 용도
> 이 프로젝트의 **상위 기획 뼈대 + 핵심 엔티티**를 그래프로 제공합니다. 디테일(실행메모·일일로그)은 각 엔티티의 참조 문서 링크를 따라가면 됩니다.
```

## 작성 가이드

### 각 섹션의 역할

| 섹션 | 역할 | 필수 |
|------|------|------|
| frontmatter | 메타데이터. `node-count`로 규모 파악 | ✅ |
| 프로젝트 개요 | LLM의 1차 필터. 질문의 관련성 판단 | ✅ |
| 📋 상위 기획 | 방향성·뼈대 엔티티. 프로젝트의 "어디로" | ✅ |
| 🎯 핵심 고려사항 | 의사결정 시 반복 참조되는 기준·제약·원칙·리스크 | ✅ |
| 👤 핵심 인물 | 영향력 있는 개인 엔티티 | 있을 때 |
| 🏢 핵심 조직 | 관계·입장이 영향 주는 조직 엔티티 | 있을 때 |
| 🌐 핵심 상황 | 의사결정에 영향 주는 외부 조건 | 있을 때 |
| 관계 맵 | 엔티티 간 기획 위계 + 영향 관계 조감도 | ✅ |
| 하위 폴더 구조 | 서브디렉토리 역할 설명 | 폴더 있을 때 |
| 미해결·갭 | BLIND_SPOT. 빠진 것 명시 | ✅ |

**해당 엔티티가 없는 카테고리**: 섹션은 유지하되 "현재 명시된 엔티티 없음"으로 표기. 섹션 자체 삭제 금지.

### 관계 유형 표기

| 표기 | 의미 | 예시 |
|------|------|------|
| →(상위)→ | A가 B의 상위 기획 | 비전 → 로드맵 |
| →(cascade)→ | A 변경 시 B 연쇄 변경 | 비전 → 로드맵 → 액션 |
| →(의사결정)→ | A가 B를 의사결정으로 이끔 | 인물 → 기획 |
| →(영향)→ | A가 B에 영향을 줌 | 상황 → 기획 |
| →(제약)→ | A가 B를 제약 | 상황 → 고려사항 |
| →(기준)→ | A가 B의 판단 기준 | 고려사항 → 기획 |
| ↔(파트너십)↔ | 협력 관계 | 조직 ↔ 기획 |
| ↔(연관)↔ | 상호 연관 | 인물 ↔ 인물 |
| ←(근거)← | A가 B의 근거·출처 | 리서치 → 기획 |

### 경량 모드 (엔티티 30개+)

- 각 엔티티 의의를 1줄로 축약
- 핵심 엔티티(현행 상위 기획·키 플레이어)만 풀스펙
- 나머지는 이름 + 1줄 의의만
- frontmatter에 `mode: lightweight` 추가

### 위키링크 규칙

- 파일명에서 `.md` 제외: `[[자벌레_통합전략_v1.0]]` ✅
- 파일명 정확 일치 필수. 추측으로 링크 생성 금지
- 하위 폴더 파일은 경로 포함: `[[_archive/구버전]]`
- 엔티티 이름 자체는 위키링크로 감쌈(그래프뷰 노드화). 참조 문서는 실제 파일명 위키링크
- PDF는 위키링크 대신 파일명만 텍스트로 기재

### 엔티티 명명 규칙

- **상위 기획**: 전략·비전·로드맵 명칭 그대로 (예: `[[자벌레_통합전략_v2.0]]`)
- **핵심 고려사항**: 원칙·제약·리스크 명 (예: `[[원칙:SoT우선]]`, `[[리스크:규제지연]]`)
- **핵심 인물**: 실명 또는 역할명 (예: `[[홍길동]]`, `[[투자자A]]`)
- **핵심 조직**: 조직명 그대로 (예: `[[KOSME]]`, `[[경쟁사B]]`)
- **핵심 상황**: 상황 명 (예: `[[상황:2026금리인상]]`, `[[상황:K-Content규제]]`)

### 갱신 시 diff 규칙

- 신규 엔티티 → 해당 카테고리 섹션에 노드 추가 + 관계 맵 갱신
- 엔티티 상태 변경 → 의의·상태·관계만 외과적 수정
- 엔티티 제거 (폐기·무관해짐) → 노드 삭제 + 관계 맵 정리
- **디테일 문서만 추가/변경**: 갱신 불필요 (INVARIANT 6)
- 변경 없는 엔티티 → 그대로 유지 (재작성 금지)
- frontmatter `updated` + `node-count` 재계산

### 안티패턴 (자주 하는 실수)

| 실수 | 왜 틀렸나 | 옳은 방식 |
|------|-----------|----------|
| 모든 .md를 노드로 나열 | 디테일이 상위 기획을 묻음 | 엔티티만 노드. 문서는 참조 링크 |
| "2026-03-15 미팅노트"를 노드화 | 단발 이벤트는 엔티티 아님 | 미팅에서 나온 결정·인물을 엔티티로 |
| 드래프트를 상위 기획 노드로 | 확정되지 않은 것은 노드 아님 | "미해결·갭" 섹션에 기록 |
| 지나가는 인물 언급을 인물 노드로 | 영향력 없는 단순 언급 | 의사결정·입장 있는 인물만 |
| 일반 배경설명을 상황 노드로 | 외부 조건이 아닌 단순 설명 | 상위 기획의 개요에 편입 |
