# 템플릿 라이브러리 + 항목별 가이드

유형 판별 후 이 문서를 로드하여 해당 유형 템플릿으로 초안을 생성한다.

---

## 유형 A: 풀 스펙 (의뢰인+프로그램 설계)

실전 근거: KISAS캠프(494줄), K-Pop Camp(282줄), KISAS유학원(71줄)에서 추출. 의뢰인 프로젝트는 좌표계·인물·전략·프로그램 구조가 필수.

### 템플릿

```markdown
# [프로젝트명] — 작업 지침

## 1. 프로젝트 좌표계
- **사업명:**
- **의뢰인:**
- **핵심 포지션:** (한 문장)
- **대상:** (타겟 고객/참가자)
- **형태:** (캠프/서비스/플랫폼 등)
- **현재 단계:**

## 2. 핵심 인물
| 인물 | 역할 | 비고 |
|------|------|------|

## 3. 사업 구조·전략 요약
- **수익 모델:**
- **경쟁 해자(Moat):**
- **핵심 갭·리스크:** (🔴로 표기)

## 4. 작업 규칙
- (프로젝트 고유 원칙)
- (용어·톤·금기 사항)
- 금액 표기: 외화 시 원화 병기

## 5. SoT & 산출물
| 산출물 | 버전 | 상태 | 비고 |
|--------|------|------|------|

## 6. 확정 수치
| 항목 | 값 | 출처 |
|------|-----|------|

## 7. 필요 산출물 로드맵
- (레이어별 필요 산출물 목록)
```

### 가이드 — 각 섹션을 채울 때

| 섹션 | 채우는 법 | 자주 빠뜨리는 것 |
|------|----------|----------------|
| §1 좌표계 | 형에게 "이 프로젝트를 한 문장으로?" 질문 | 유사 프로젝트와의 차이점 (K-Pop Camp↔KISAS 처럼) |
| §2 인물 | 인터뷰 A2 답변으로 채움. 의사결정권자 + 실무 담당자 최소 2명 | 학력·경력 중 프로젝트 관련 자격. 답변이 있으면 [미확정] 금지 |
| §3 전략 | 수익모델·해자·리스크 3요소 | 리스크를 긍정적으로만 적는 실수 |
| §4 규칙 | "이 프로젝트에서만 적용되는 특수한 것" | 상위 CLAUDE.md 참조 관계 |
| §5 SoT | 현재 존재하는 파일을 개별 행으로 기재. 묶음 처리(예: "WF 6종") 금지 — 각 파일을 별도 행에 | 버전 불일치 (파일명 vs 기재) |
| §6 수치 | 출처 필수 (문서명+페이지) | Python 검증 대상 표기 |
| §7 로드맵 | 레이어별 나열 (미착수도 포함) | "지금은 필요 없으니까" 하고 생략 |

## 유형 B: 표준형 (리서치→기획 파이프라인)

실전 근거: C8공통(79줄), CEO파이프라인(40줄), 인천(31줄), 화성(83줄), 두나무(61줄)에서 추출. Phase 기반 작업에서 상태 추적이 핵심.

### 템플릿

```markdown
# [프로젝트명] — 작업 지침

## 1. 현재 상태
- **단계:**
- **완료:**
- **진행중:**
- **미착수:**

## 2. 작업 규칙
- (프로젝트 고유 원칙)
- (Phase 순서, 용어, 금기)
- 금액 표기: 외화 시 원화 병기

## 3. SoT & 산출물
| 산출물 | 버전 | 상태 | 비고 |
|--------|------|------|------|

## 4. 확정 수치
| 항목 | 값 | 출처 |
|------|-----|------|
```

### 가이드

| 섹션 | 채우는 법 | 자주 빠뜨리는 것 |
|------|----------|----------------|
| §1 상태 | Phase 0~3 중 현재 위치 명시 | "진행중"과 "미착수" 구분 |
| §2 규칙 | Phase 순서 준수 규칙 + SoT 원본 취급 규칙 | 편향방지 원칙 (두나무 사례) |
| §3 SoT | 소스(입력)와 산출물(출력) 구분 | _sources/ 파일 목록 누락 |
| §4 수치 | 핵심 수치만 (전체 복사 금지) | 출처 페이지 번호 |

## 유형 C: 최소형 (단건 분석/판단)

실전 근거: MBMG(25줄), AI기획설계(19줄), BP(18줄), 자벌레(36줄)에서 추출. 빠르게 착수하되 최소 맥락은 확보.

### 템플릿

```markdown
# [프로젝트명] — 작업 지침

## 1. 현재 상태
- **단계:**
- **결론:** (있으면)

## 2. 작업 규칙
- (핵심 1~3개)

## 3. SoT & 산출물
| 산출물 | 버전 | 상태 | 비고 |
|--------|------|------|------|
```

### 가이드

| 섹션 | 채우는 법 | 자주 빠뜨리는 것 |
|------|----------|----------------|
| §1 상태 | 완료된 건이면 결론 명시 (MBMG "제휴불가" 처럼) | 종결 판정 후 "다음 액션" |
| §2 규칙 | 3개 이내로 압축 | 상위 폴더 참조 관계 |
| §3 SoT | 파일 있는 것만 | _archive/ 존재 여부 |

## 상위 CLAUDE.md 참조 패턴

C8 프로젝트군처럼 공통 지침이 있는 경우:

```markdown
## 2. 작업 규칙
- 루트 지침: [[00 C8 공통/CLAUDE.md]] 참조
- (이 프로젝트 고유 규칙만 아래에 기재)
```

**판별**: 같은 접두어 그룹(00 C8 *, 01 피디님_* 등)에 공통 CLAUDE.md가 있으면 참조 삽입.
