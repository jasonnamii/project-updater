# 전체 갱신 절차 (볼트 전체 순회)

"전체 프로젝트 업데이트 해줘" 등 볼트 전체 대상 요청 시 실행.

```
①프로젝트 폴더 목록 수집 → ②병렬 처리(Agent tool) → ③전체 리포트
```

## ① 프로젝트 폴더 목록 수집

볼트 루트의 프로젝트 폴더(00~, 01~, 02~ 등) 전체 목록을 수집한다. 현재 기준 약 21개.

## ② 프로젝트별 처리 (Agent tool 병렬)

**Agent tool로 3~5개 프로젝트를 동시 처리.** 각 Agent에게 프로젝트 폴더 경로 + 아래 절차를 위임한다.

| 단계 | 행동 | 비고 |
|------|------|------|
| A. CLAUDE.md 확인 | 존재 → 건강검진 / 미존재 → 생성 | 생성은 claude-md-generation.md |
| B. GRAPH.md 확인 | 존재 → 갱신 / 미존재 → 생성 | graph-md-generation.md |

### A. CLAUDE.md 건강검진 (존재 시)

읽기 전용. 아래 5항목을 체크하여 이슈만 리포트한다. **자동 수정 금지 — 갱신은 형 별도 승인 시 UP §H 규칙으로.**

| # | 체크 항목 | 판정 |
|---|----------|------|
| 1 | `[미확정]` 잔존 개수 | 5개 초과 → ⚠ |
| 2 | SoT 테이블 파일이 실제 존재하는가 | 누락 → ⚠ |
| 3 | 확정 수치에 출처가 있는가 | 출처 없음 → ⚠ |
| 4 | 상위 CLAUDE.md 참조가 필요한데 빠졌는가 | 같은 접두어 그룹에 공통 CLAUDE.md 있으면 참조 필수 |
| 5 | 200줄 이내인가 | 초과 → ⚠ |

### A. CLAUDE.md 생성 (미존재 시)

claude-md-generation.md 절차를 따르되, 전체 갱신 모드에서는 **폴더 스캔 + 최소 추론으로 C유형 초안를 자동 생성**한다. 형에게 일괄 검토용으로 제시. 인터뷰 생략.

### B. GRAPH.md 갱신 (존재 시)

graph-md-generation.md §갱신 규칙 적용.

### B. GRAPH.md 생성 (미존재 시)

graph-md-generation.md 생성 절차 적용. 폴더 스캔 → 문서 읽기 → 초안 생성.

## ③ 전체 리포트

모든 프로젝트 처리 완료 후, 아래 형식으로 요약 테이블을 제시한다.

```
| 프로젝트 | CLAUDE.md | GRAPH.md | 이슈 |
|----------|-----------|----------|------|
| 00 C8 공통 | ✅ 정상 | ✅ 갱신 완료 | — |
| 00 C8 앱_UI설계 | ⚠ [미확정] 7개 | ✅ 갱신 완료 | SoT 2건 누락 |
| 01 피디님_기타 | 🆕 C유형 초안 생성 | ✅ 갱신 완료 | 형 검토 필요 |
```

**🆕 표시 프로젝트는 형의 일괄 검토·승인 후 저장.** 승인 전까지 임시 보관.
