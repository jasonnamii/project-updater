# Project Updater

프로젝트 CLAUDE.md(작업지침) + GRAPH.md(지식그래프) 통합 엔진. CLAUDE.md는 3유형 템플릿 기반, GRAPH.md는 Obsidian 위키링크 기반 지식그래프. LLM 지시 + 파일 도구로 완결.

---

## 왜 이 스킬이 필요한가

- **CLAUDE.md 없으면** 매 세션마다 맥락을 처음부터 설명해야 한다
- **GRAPH.md 없으면** "이 폴더에 뭐가 있는지" 파악하려고 매번 전체 파일을 읽어야 한다
- 이 스킬은 두 문제를 동시에 해결: CLAUDE.md(작업 규칙·SoT·수치) + GRAPH.md(문서 관계·핵심개념·요약)

---

## 모드 판정 — 발동 시 1회 확정

| 조건 | 모드 | 실행 |
|------|------|------|
| CLAUDE.md 없음 | **CLAUDE.md 생성** | 아래 §CLAUDE.md 생성 |
| GRAPH.md 없음 | **GRAPH.md 생성** | 아래 §GRAPH.md 생성 |

| "프로젝트 업데이트" 요청 | **전체 갱신** | 아래 §전체 갱신 절차 (볼트 전체 순회) |

## CLAUDE.md 생성

**폴더 스캔 통합:** CLAUDE.md + GRAPH.md 동시 요청 시 폴더 스캔 1회만 실행. 스캔 결과를 공유하여 CLAUDE.md 생성 + GRAPH.md 생성에 재사용. 순차 요청이더라도 같은 세션 내면 이전 스캔 결과 캐시 활용.

```
①인터뷰(유형판별+정보수집) → ②템플릿 로드 → ③폴더 스캔(선택) → ④초안 생성 → ⑤형 검토·수정 → ⑥저장
```

### ① 인터뷰

**발동 즉시 인터뷰 프로토콜을 실행.** `→ references/interview.md 참조`

- **1단계**: AskUserQuestion 1회로 유형 판별 3개 질문 동시 수집
- **2단계**: 유형별 필수 질문 (A=4개, B=2개, C=2개) 수집 → 필수 답변 모이면 즉시 초안 생성
- **"그냥 빨리 만들어"**: 폴더 스캔 + 최소 추론으로 C유형 초안 즉시 생성

### ② 템플릿 로드

`→ references/templates.md 참조`

### ③ 폴더 스캔 (선택)

마운트된 프로젝트 폴더가 있으면: 파일 목록·frontmatter·_sources/ 수집 → SoT 섹션 자동 채움

### ④ 초안 생성

| 규칙 | 이유 |
|------|------|
| 인터뷰 답변 있는 항목 반드시 채움 | [미확정] 남기면 인터뷰 의미 없음 |
| 추론 불가 항목만 `[미확정]` | 빈 플레이스홀더 금지 |
| 가정 가능하면 가정 명시 후 채움 | 빈 칸보다 가정+채움이 낫다 |
| 200줄 이내 | A4 2장 한도 |
| `[미확정]` 5개 초과 시 경고 | 양식이 아니라 지침이어야 한다 |
| 파일명: `CLAUDE.md` 고정 | 프로젝트 폴더 루트 |

### ⑤ 형 검토 → ⑥ 저장

형 승인 후 저장. 저장 후 1줄 경로 보고 필수.

## GRAPH.md 지식그래프 생성

```
①폴더 스캔 → ②문서 읽기(내용 분석) → ③그래프 초안 생성 → ④형 검토·수정 → ⑤저장
```

### ① 폴더 스캔

프로젝트 폴더의 전체 파일 목록 수집. .md 파일 우선, PDF·기타는 파일명만.

### ② 문서 읽기

**각 .md 파일의 내용을 실제로 읽는다.** (CLAUDE.md 생성의 폴더 스캔과 핵심 차이 — 파일명이 아닌 내용 분석)
- 문서별 3-5줄 요약 추출
- 핵심 개념·용어 식별
- 문서 간 관계 파악 (상위/하위, 근거, 참조, 선후, 통합)

**대형 폴더 (문서 20개+)**: Agent tool로 병렬 읽기 권장.

### ③ 그래프 초안 생성

`→ references/graph-template.md 참조`

| 규칙 | 이유 |
|------|------|
| Obsidian `[[위키링크]]` 사용 | 그래프뷰 연동 + LLM 컨텍스트 효율 |
| 문서별 요약 3-5줄 + 핵심개념 + 상태 + 관계 | 풀스펙 |
| 관계 맵 섹션으로 전체 조감도 | 개별 노드 + 조감도 이중 제공 |
| 미해결·갭 섹션 필수 | BLIND_SPOT 원칙 |
| 파일명: `GRAPH.md` 고정 | 프로젝트 폴더 루트 |

**경량 모드 (문서 50개+)**: 요약 2줄 축약, 핵심 문서만 풀스펙. node-count > 50이면 자동 제안.

### ④ 형 검토 → ⑤ 저장

형 승인 후 저장.

## GRAPH.md 갱신

| 트리거 | 행동 |
|--------|------|
| "GRAPH.md 갱신" 요청 | 기존 읽기 → 폴더 재스캔 → 신규/삭제/변경 식별 → diff 반영 |
| 문서 추가/삭제 감지 | "GRAPH.md 갱신 필요" 1줄 제안 (자동 실행 안 함) |
| CLAUDE.md 구조 변경 | 프로젝트 개요 섹션 갱신 필요 여부 확인 |

**갱신 규칙:** 기존 요약·관계 유지, 변경분만 외과적 반영. frontmatter `updated` + `node-count` 재계산.

---

## 전체 갱신 절차

"전체 프로젝트 업데이트 해줘" 등 볼트 전체 대상 요청 시 실행.

```
①프로젝트 폴더 목록 수집 → ②병렬 처리(Agent tool) → ③전체 리포트
```

### ① 프로젝트 폴더 목록 수집

볼트 루트의 프로젝트 폴더(00~, 01~, 02~ 등) 전체 목록을 수집한다. 현재 기준 약 21개.

### ② 프로젝트별 처리 (Agent tool 병렬)

**Agent tool로 3~5개 프로젝트를 동시 처리.** 각 Agent에게 프로젝트 폴더 경로 + 아래 절차를 위임한다.

| 단계 | 행동 | 비고 |
|------|------|------|
| A. CLAUDE.md 확인 | 존재 → 건강검진 / 미존재 → 생성 | 생성은 §CLAUDE.md 생성 절차 |
| B. GRAPH.md 확인 | 존재 → 갱신 / 미존재 → 생성 | §GRAPH.md 생성·갱신 절차 |

**A. CLAUDE.md 건강검진** (존재 시):

읽기 전용. 아래 5항목을 체크하여 이슈만 리포트한다. **자동 수정 금지 — 갱신은 형 별도 승인 시 UP §H 규칙으로.**

| # | 체크 항목 | 판정 |
|---|----------|------|
| 1 | `[미확정]` 잔존 개수 | 5개 초과 → ⚠ |
| 2 | SoT 테이블 파일이 실제 존재하는가 | 누락 → ⚠ |
| 3 | 확정 수치에 출처가 있는가 | 출처 없음 → ⚠ |
| 4 | 상위 CLAUDE.md 참조가 필요한데 빠졌는가 | 같은 접두어 그룹에 공통 CLAUDE.md 있으면 참조 필수 |
| 5 | 200줄 이내인가 | 초과 → ⚠ |

**A. CLAUDE.md 생성** (미존재 시):

§CLAUDE.md 생성 절차를 따르되, 전체 갱신 모드에서는 **폴더 스캔 + 최소 추론으로 C유형 초안를 자동 생성**한다. 형에게 일괄 검토용으로 제시. 인터뷰 생략.

**B. GRAPH.md 갱신** (존재 시): §GRAPH.md 갱신 규칙 적용.

**B. GRAPH.md 생성** (미존재 시): §GRAPH.md 지식그래프 생성 절차 적용. 폴더 스캔 → 문서 읽기 → 초안 생성.

### ③ 전체 리포트

모든 프로젝트 처리 완료 후, 아래 형식으로 요약 테이블을 제시한다.

```
| 프로젝트 | CLAUDE.md | GRAPH.md | 이슈 |
|----------|-----------|----------|------|
| 00 C8 공통 | ✅ 정상 | ✅ 갱신 완료 | — |
| 00 C8 앱_UI설계 | ⚠ [미확정] 7개 | ✅ 갱신 완료 | SoT 2건 누락 |
| 01 피디님_기타 | 🆕 C유형 초안 생성 | ✅ 갱신 완료 | 형 검토 필요 |
```

**🆕 표시 프로젝트는 형의 일괄 검토·승인 후 저장.** 승인 전까지 임시 보관.

## Gotchas

- **유형 과잉 판정**: 단건 분석인데 A유형으로 만들면 빈 섹션 투성이. C로 시작.
- **폴더 스캔 과신**: 파일명만으로 내용 추측 금지. 존재 여부만 반영.
- **상위 참조 누락**: C8 하위에서 `00 C8 공통/CLAUDE.md` 참조 빠뜨리면 불일치.
- **플레이스홀더 남발**: `[미확정]` 10개 넘으면 양식이지 지침이 아님.
- **200줄 초과**: A유형 위험. 로드맵은 레이어명만, 상세는 별도 문서로.
- **GRAPH.md 위키링크 불일치**: `[[문서명]]`이 실제 파일명과 다르면 broken link. 정확한 파일명(.md 제외) 필수.
- **GRAPH.md 과다 토큰**: 문서 50개+ 폴더에서 풀스펙 → 300줄+. 경량 모드 자동 전환.
- **GRAPH.md 갱신 시 덮어쓰기**: 변경 없는 문서 요약은 유지. 전체 재작성 금지.
- **CLAUDE.md 없이 GRAPH.md만 요청**: 가능. 독립적으로 생성·갱신 가능.
